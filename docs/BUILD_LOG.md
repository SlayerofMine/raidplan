RaidPlans is a self-hosted WoW raid/arena planner (canvas editor + PowerPoint-style step animations) built per `IMPLEMENTATION_PLAN.md`. pnpm workspace monorepo: `apps/web` (Vite+React+TS), `apps/api` (Hono), `packages/shared` (zod Plan schema + pure state-resolution). Each phase is committed separately and must be a standalone compilable repo.

**Convention:** every completed phase = one git commit; run typecheck+lint+test+build green before committing. Prettier ignores `IMPLEMENTATION_PLAN.md` (keep spec verbatim). Husky pre-commit runs lint-staged. `main` branch. Vite pinned to ^5 to match vitest 2.1 (vite 6 caused a plugin type mismatch).

Key gotchas learned:

- Konva imports its **Node build** (needs native `canvas`) under vitest/jsdom ‚Üí never import the editor tree in a jsdom test; test pure logic + HTML leaf components instead.
- Playwright browser now WORKS in this env (user fixed system libs). 8 e2e specs in `apps/web/e2e/` pass locally via `pnpm --filter @raidplan/web test:e2e` (commit e65a947). CI has a dedicated `e2e` job. E2E asserts DOM-observable signals (testids: object-count, zoom-level, prop-x/prop-y, properties, no-selection, canvas-container) since canvas pixels aren't queryable; real canvas drag verified via mouse down/move/up at container centre.
- Pure coord math (native‚Üîscreen, fit, zoom-to-cursor) lives in `apps/web/src/editor/canvas/coords.ts` ‚Äî unit-tested, framework-free.
- **jsdom has no ResizeObserver and loads no images.** `apps/web/src/test/setup.ts` installs a ResizeObserver mock reporting a fixed 240x480 viewport (the virtualized palette renders nothing without it). Never assert on image `load` in jsdom ‚Äî test the cache primitive (`getImageElement`) instead, and leave rendering to e2e.
- zundo config that matters: `partialize` to the PlanDoc fields only + `equality: shallow`. Without the equality check, selection/camera sets would each push a no-op history entry and undo would appear to do nothing.
- Store split: document (PlanDoc: id/title/raid/background/objects/objectIds/steps) = persisted + undoable; selection/view/stageSize/grid/clipboard/currentStepIndex = ephemeral.
- **Steps model (Phase 3, important):** `currentStepIndex === -1` (BASE_STEP_INDEX) edits `object.base`; `>= 0` writes into `steps[i].overrides`. All routed through `writeOverridable()`, which splits patches ‚Äî transforms follow the step, tint/label/z always base. Canvas AND properties panel must render the **resolved** state (`selectObjectState` + `useShallow`), never `object.base`, or overrides look like they snap back.
- **structuredClone throws on immer drafts** (Proxies). Inside `set()`, read via `get()` first if you need to clone.
- **tsup keeps `dependencies` external** ‚Üí `@raidplan/shared` (TS sources) wasn't bundled and the built API died with ERR_MODULE_NOT_FOUND at runtime while every unit test passed. `noExternal: [/^@raidplan\//]` in apps/api/tsup.config.ts fixes it. **Always smoke-test the built artifact, not just tests.**
- SQLite `foreign_keys` is OFF per-connection by default ‚Äî `openDb` must set the pragma or every `references()` is decorative.
- **zod 4 workspace-wide** (commit dcc2601). Migrated off zod 3 so better-auth can track its current release; the earlier pin + `better-call` override are GONE (don't reintroduce them). Migration was ~free: `.finite()`, `z.record(k,v)`, `z.coerce.number()`, `error.issues[].path` all unchanged; only `z.string().url()` ‚Üí `z.url()`. **Keep ONE zod major across shared/api/web** ‚Äî a split means two incompatible `z.infer` types with identical names. Stack: zod 4.4, better-auth 1.6, drizzle-orm 0.45, drizzle-kit 0.31, better-sqlite3 12.
- API auth seam: `AppDeps.getUserId(req) ‚Üí userId` + `viewerFor(db, userId) ‚Üí Viewer`. Tests drive the API as any user via `createApp({ getUserId: () => "u1" })` ‚Äî no OAuth needed.
- GSAP can't tween Konva's method-style accessors (`node.x()`), so `compileStep` tweens a plain proxy and pushes via `apply` ‚Äî that's also why it's unit-testable without a canvas. Use ONE proxy per object or concurrent animations clobber each other. It returns `initial` (start state adjusted for entrance effects) which the engine snaps to before playing.
